## react-shop 도메인 구조

### 1) 현재 코드 구조
- `src/app/`: 라우팅과 페이지 진입점을 관리한다.
- `src/domains/`: 도메인별 화면, 훅, 서비스, 타입을 우선 배치하는 위치다.
- `src/shared/`: 여러 도메인이 공통으로 사용하는 컴포넌트, 유틸, 상수, 타입을 관리한다.
- `src/middleware.ts`: 공통 미들웨어를 관리한다.

### 2) 도메인 책임 기준
- 신규 또는 수정 파일은 가능한 한 `src/domains/{도메인}` + `src/shared/{공통}` 구조를 우선 적용한다.
- 조회 가능한 데이터는 하드코딩하지 않고 API나 DB 조회 결과를 사용한다.
- 중복 서버 호출 로직은 공통 호출 유틸 사용을 우선 검토한다.
- 동일 역할 로직이 여러 곳에 있으면 공통화 가능한 최소 단위로 통합한다.

### 3) 구현 기준
- 아이콘은 현재 코드 기준의 `Font Awesome Kit` 사용 여부를 먼저 확인한다.
- 슬라이더는 현재 사용 중인 `swiper`와 `swiper/react` 기준을 우선 확인한다.
- `<img>` 사용으로 ESLint 경고가 발생하면 `next/image` 전환을 우선 검토한다.
- React 훅 규칙 경고나 오류는 수정 범위 내에서 즉시 해소한다.

### 4) 경계 규칙
- 프론트 `/api` 경로는 임의로 바꾸지 않는다.
- API 추가, 변경, DB 연동은 `spring-back-end`에서 처리한다.
- 상세 도메인 규칙은 향후 본 문서 하위 문서로 확장한다.
